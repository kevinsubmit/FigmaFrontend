# 开发更新 2026-04-13：生产 Docker 部署模板

## 目标

为当前项目补一套可落地的生产部署模板，避免继续沿用开发态运行方式：

- 不再依赖本机直接跑 `uvicorn`
- 不再依赖 Vite dev server
- 不再把源码目录挂进生产容器
- 统一通过 Docker 服务编排部署

## 生产结构

新增的生产模板采用以下结构：

1. `nginx`
   - 对外监听 `80/443`
   - 负责 HTTPS 终止和反向代理
   - 按域名分发到 app/admin/api

2. `frontend-web`
   - H5 前端静态构建
   - 由容器内 Nginx 提供静态文件

3. `admin-web`
   - 后台管理端静态构建
   - 由容器内 Nginx 提供静态文件

4. `backend`
   - FastAPI API 服务

5. `backend-scheduler`
   - 独立 scheduler worker

6. `db`
   - MySQL 8

7. `redis`
   - Redis 7

## 约定的生产域名

建议使用三套独立域名：

- `APP_DOMAIN=app.nailsdash.app`
- `ADMIN_DOMAIN=admin.nailsdash.app`
- `API_DOMAIN=api.nailsdash.app`

这样三端客户端和管理后台都可以固定连正式 API 域名，不再依赖局域网 IP。

## 部署步骤

### 1. 准备环境文件

复制模板：

```bash
cp .env.prod.example .env.prod
cp backend/.env.prod.example backend/.env.prod
```

至少修改：

- 域名
- TLS 证书文件名
- MySQL 密码
- `SECRET_KEY`
- `DATABASE_URL`
- `CORS_ORIGINS`

注意：

- `MYSQL_PASSWORD` 必须和 `backend/.env.prod` 里的 `DATABASE_URL` 密码保持一致

### 2. 准备 TLS 证书

把证书放到：

- `deploy/certs/fullchain.pem`
- `deploy/certs/privkey.pem`

或在 `.env.prod` 里改成你自己的证书文件名。

### 3. 启动服务

```bash
docker compose --env-file .env.prod -f docker-compose.prod.yml up -d --build
```

### 4. 跑迁移

```bash
docker compose --env-file .env.prod -f docker-compose.prod.yml exec -T backend alembic upgrade head
```

### 5. 检查状态

```bash
docker compose --env-file .env.prod -f docker-compose.prod.yml ps
```

确认：

- `db` healthy
- `redis` healthy
- `backend` up
- `backend-scheduler` up
- `frontend-web` up
- `admin-web` up
- `nginx` up

### Admin Dashboard

`admin-dashboard/Dockerfile.prod`：

- Node 构建阶段执行 `npm ci`
- 生成 `dist/`
- 第二阶段用 Nginx 提供静态文件

## 与开发环境的区别

开发版 `docker-compose.yml`：

- 挂源码
- 前端跑 Vite dev server
- 更适合本地调试

生产版 `docker-compose.prod.yml`：

- 不挂源码
- 前端和后台管理端都使用静态构建产物
- `backend` 和 `backend-scheduler` 明确分离
- 只通过边缘 `nginx` 对外暴露

## 当前边界

这次提供的是可直接使用的生产模板，已经补上：

- `systemd` 管理 Docker Compose
- `Certbot` 首次签发脚本
- `Certbot renew` 定时续签
- 证书同步到容器挂载目录
- 证书续签后的 Nginx reload

仍未包含：

- 自动数据库备份
- 自动镜像发布流水线
- 多机部署 / Kubernetes

## 上线推荐顺序

### 1. 安装 systemd 单元

```bash
sudo ./deploy/scripts/install-systemd.sh /opt/nailsdash
```

### 2. 准备环境文件

- `/opt/nailsdash/.env.prod`
- `/opt/nailsdash/backend/.env.prod`
- `/etc/nailsdash/nailsdash.env`

### 3. 启动生产服务

```bash
sudo systemctl enable --now nailsdash-compose.service
```

### 4. 首次签发证书

```bash
cd /opt/nailsdash
sudo ./deploy/scripts/bootstrap-letsencrypt.sh
```

### 5. 开启自动续签

```bash
sudo systemctl enable --now nailsdash-certbot-renew.timer
```

### 6. 续签验证

```bash
sudo systemctl start nailsdash-certbot-renew.service
sudo systemctl status nailsdash-certbot-renew.service
```

## 说明

- `bootstrap-letsencrypt.sh` 使用 `certbot certonly --standalone`
- 首次签发时会暂时停掉生产 `nginx` 容器，再完成签发与证书同步
- 之后续签通过 host 的 `certbot renew` 完成，并在 deploy hook 中：
  1. 复制新证书到 `deploy/certs`
  2. 对容器内 `nginx` 执行 reload
