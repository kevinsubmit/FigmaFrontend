# 开发更新（2026-03-02）：H5 / iOS 会话自动续签与分享弹窗修复

## 本次目标
- 解决前台 H5 与 iOS 端频繁出现 `session expired`、30 分钟后直接被踢回登录的问题。
- 保留安全策略：当刷新令牌失效、账号受限或鉴权失败时，仍然强制回登录。
- 修复 iOS `Refer a Friend` 页面首次点击分享按钮不弹窗的问题。

## 关键改动

### 1) H5：401 自动刷新 Access Token 后重试原请求
文件：`frontend/src/lib/api.ts`

- 响应拦截器新增逻辑：
  - 当请求返回 401 且不是登录/注册/验证码/refresh 接口时，先调用 `/auth/refresh`。
  - 刷新成功后，使用新 `access_token` 自动重放一次原请求。
  - 刷新失败才执行 `forceRelogin()` 清理会话并跳转登录页。
- 增加 refresh 并发去重：多个请求同时 401 时只发起一次 refresh，请求共享结果。
- 刷新成功后清理 GET 内存缓存与 in-flight 请求，避免旧 token 缓存污染。

### 2) iOS：APIClient 内统一实现自动续签
文件：`ios-app/NailsDashIOS/Sources/Core/Network/APIClient.swift`

- 在 `APIClient.request` 内部改为 `requestInternal`：
  - 401 时尝试 `POST /auth/refresh?refresh_token=...`。
  - 刷新成功后自动携带新 token 重试当前请求一次。
  - 若刷新失败，继续走现有全局未授权处理（触发 AppState 强制回登录）。
- 增加 `TokenRefreshCoordinator`（actor）：
  - 并发 401 只触发一次刷新任务，避免刷新风暴。
- 不对登录/注册/验证码/refresh 自身请求做自动刷新，避免死循环。

### 3) 后端：`/auth/refresh` 入参兼容增强
文件：`backend/app/api/v1/endpoints/auth.py`

- 原先仅支持 query 参数 `refresh_token`。
- 现改为同时支持：
  - Query：`/auth/refresh?refresh_token=...`
  - Body：`{"refresh_token":"..."}`
- 若两者都没有，返回明确错误：`refresh_token is required`。

### 4) 前端 Auth API 统一 refresh 调用方式
文件：`frontend/src/api/auth.ts`

- `refreshToken()` 调整为 query 参数调用，与后端接口当前规范保持一致。

### 5) iOS 分享弹窗首次点击修复
文件：`ios-app/NailsDashIOS/Sources/Features/Home/HomeView.swift`

- `ReferAFriendView` 的 `.sheet` 触发由布尔状态改为 `Identifiable` payload 绑定。
- 修复首次点击按钮时 sheet 未弹出的时序问题。

## 当前登录态行为（前台）
- Access Token 过期时间：30 分钟（后端配置）。
- Refresh Token 过期时间：30 天（后端配置）。
- 现在前台行为变更为：
  - Access 过期后优先自动刷新并继续当前操作。
  - 仅当 Refresh 失效/缺失，或账号被限制/禁用时，才跳回登录页。

## 风险与说明
- 本次未改变登录安全策略本身，仅改变“过期后的用户体验路径”（先自动续签再决定是否登出）。
- 若后续需要进一步提升体验，可增加“过期前静默续签”策略（例如 access token 即将过期前主动刷新）。
