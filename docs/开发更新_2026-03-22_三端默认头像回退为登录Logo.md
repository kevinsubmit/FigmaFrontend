# 开发更新 2026-03-22：三端默认头像回退为登录 Logo

## 目标

当用户没有设置头像时，Android、iOS 和 H5 统一使用登录页面顶部的 logo 作为默认头像，避免三端默认占位样式不一致。

## 本次改动

### Android

- 新增通用默认头像组件 `AuthLogoAvatar`
- `Profile` 页面头像为空时，回退到登录页同款 logo
- 资料设置页头像为空时，回退到登录页同款 logo

涉及文件：

- `android-app/app/src/main/java/com/nailsdash/android/ui/component/AuthLogoAvatar.kt`
- `android-app/app/src/main/java/com/nailsdash/android/ui/screen/ProfileScreen.kt`
- `android-app/app/src/main/java/com/nailsdash/android/ui/screen/SettingsScreens.kt`

### iOS

- 将登录页顶部 logo 抽为可复用组件 `AuthLogoBadgeView`
- `Profile` 页面头像为空时，回退到登录页同款 logo
- 设置页头像为空时，回退到登录页同款 logo

涉及文件：

- `ios-app/NailsDashIOS/Sources/Features/Auth/LoginView.swift`
- `ios-app/NailsDashIOS/Sources/Features/Profile/ProfileCenterView.swift`
- `ios-app/NailsDashIOS/Sources/Features/Settings/SettingsModule.swift`

### H5

- 新增通用 logo 组件 `AuthLogoBadge`
- 登录页和注册页顶部统一复用该组件
- `Profile` 页面头像为空时，回退到登录页同款 logo
- 编辑资料页头像为空时，回退到登录页同款 logo

涉及文件：

- `frontend/src/components/AuthLogoBadge.tsx`
- `frontend/src/components/Login.tsx`
- `frontend/src/components/Register.tsx`
- `frontend/src/components/Profile.tsx`
- `frontend/src/components/EditProfile.tsx`

## 范围说明

这次只处理“当前登录用户”的头像回退：

- Profile 首页
- 资料设置 / 编辑资料

未修改第三方头像场景，例如：

- 评论区用户头像
- 技师头像
- 其他用户资料卡片

## 验证

- Android：`./gradlew :app:assembleDebug -x lint`
- iOS：`xcodebuild -project ios-app/NailsDashIOS.xcodeproj -scheme NailsDashIOS -configuration Debug -destination 'generic/platform=iOS Simulator' CODE_SIGNING_ALLOWED=NO build`
- H5：`npm run build`

三项均已通过。
