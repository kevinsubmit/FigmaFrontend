# Android 状态栏时间与主页面底部留白修复

## 本次调整
- 修复 Android 顶部状态栏时间不可见问题，强制系统栏使用浅色图标。
- 保持状态栏与导航栏透明，不改页面整体 edge-to-edge 行为。
- 统一 `Book / Deals / Appointments` 三个主页面底部内容留白，避免底部黑色区域视觉高度比 `Home / Profile` 更高。

## 影响文件
- `android-app/app/src/main/java/com/nailsdash/android/MainActivity.kt`
- `android-app/app/src/main/java/com/nailsdash/android/ui/screen/StoresScreen.kt`
- `android-app/app/src/main/java/com/nailsdash/android/ui/screen/DealsScreen.kt`
- `android-app/app/src/main/java/com/nailsdash/android/ui/screen/AppointmentsScreen.kt`

## 具体修改
- `MainActivity`：
  - `window.statusBarColor = Color.TRANSPARENT`
  - `window.navigationBarColor = Color.TRANSPARENT`
  - `isAppearanceLightStatusBars = false`
  - `isAppearanceLightNavigationBars = false`
- `StoresScreen`：底部 `contentPadding.bottom` 从 `26.dp` 收到 `14.dp`
- `DealsScreen`：底部 `contentPadding.bottom` 从 `26.dp` 收到 `14.dp`
- `AppointmentsScreen`：外层内容 `bottom padding` 从 `28.dp` 收到 `14.dp`

## 验证
- 执行：`JAVA_HOME='/Applications/Android Studio.app/Contents/jbr/Contents/Home' ./gradlew :app:assembleDebug -x lint`
- 结果：`BUILD SUCCESSFUL`

## 说明
- 这次没有再改根 `Scaffold` 的系统 inset，避免主页面顶部内容顶到状态栏。
- 如果后续 `Book / Deals / Appointments` 仍有视觉高度差异，下一步应继续收各自 header 的高度，而不是再碰全局系统栏配置。
