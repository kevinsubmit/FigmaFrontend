# 三端 Settings 联系方式后台配置化

## 目标
- 把前端 `Settings` 页面里 `Feedback & Support`、`Partnership Inquiry` 的联系方式从硬编码改为后台配置。
- 在后台管理系统新增超级管理员配置页，统一维护三端联系方式。

## 后端
- 新增 `support_contact_settings` 单例配置表。
- 新增公开接口：`GET /api/v1/support-contact`
- 新增后台接口：
  - `GET /api/v1/support-contact/admin`
  - `PUT /api/v1/support-contact/admin`
- 配置字段：
  - `feedback_whatsapp_url`
  - `feedback_imessage_url`
  - `feedback_instagram_url`
  - `partnership_whatsapp_url`
  - `partnership_imessage_url`
- 后端校验只允许以下前缀：
  - `http://`
  - `https://`
  - `sms:`
- 初始化默认值延续当前线上硬编码：
  - `https://wa.me/14151234567`
  - `sms:+14151234567`
  - `https://instagram.com`

## 后台管理系统
- 新增页面：`/admin/contact-config`
- 入口放在 `More` 页面。
- 仅超级管理员可访问。
- 支持读取和保存三端 Settings 联系方式配置。

## iOS / Android / H5
- `Feedback & Support` 页面改为读取 `/support-contact`。
- `Partnership Inquiry` 页面改为读取 `/support-contact`。
- 请求失败时继续使用默认联系方式，不阻断页面使用。
- H5 对 `sms:` 使用 `window.location.href`，其余链接继续新开页。

## 验证
- Backend：`python3 -m compileall backend/app backend/alembic`
- Migration：`docker compose exec -T backend alembic upgrade head`
- Public API：`curl http://localhost:8000/api/v1/support-contact`
- Admin Dashboard：`cd admin-dashboard && npm run build`
- H5：`cd frontend && npm run build`
- Android：`cd android-app && JAVA_HOME='/Applications/Android Studio.app/Contents/jbr/Contents/Home' ./gradlew :app:assembleDebug -x lint`
- iOS：`xcodebuild -project ios-app/NailsDashIOS.xcodeproj -scheme NailsDashIOS -configuration Debug -destination 'generic/platform=iOS Simulator' CODE_SIGNING_ALLOWED=NO build`
