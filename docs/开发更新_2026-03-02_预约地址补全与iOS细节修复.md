# 开发更新（2026-03-02）：预约地址补全与 iOS 细节修复

## 本次目标
- `H5` 与 `iOS` 的预约列表、预约详情统一展示完整店铺地址（含 `zip code`）。
- 修复 iOS 若干体验细节：预约完成后 Deals 路由残留、设置页图标兜底、店铺列表默认排序逻辑等。

## 变更内容

### 1) H5：Appointments 列表与详情地址补全（含 zip）
- 文件：
  - `frontend/src/api/stores.ts`
  - `frontend/src/components/Appointments.tsx`
  - `frontend/src/components/AppointmentDetailsDialog.tsx`
- 关键调整：
  - `Store` 类型补充 `city/state/zip_code` 字段。
  - 新增统一地址拼接逻辑：`street, city, state zip`。
  - 预约列表地址展示优先级：
    1. 预约内 `store` 对象完整地址
    2. 店铺列表回填地址（按 `store_id`）
    3. 原始 `store_address`
  - 预约详情弹窗地址展示与地图查询改为完整地址，并允许多行显示，避免 zip 被截断。
  - 详情弹窗按 `store_id` 重新拉店铺详情，避免切换订单时复用旧店铺地址。

### 2) iOS：Appointments 列表与详情地址补全（含 zip）
- 文件：
  - `ios-app/NailsDashIOS/Sources/Features/Appointments/BookAppointmentViewModel.swift`
  - `ios-app/NailsDashIOS/Sources/Features/Appointments/MyAppointmentsView.swift`
- 关键调整：
  - `MyAppointmentsViewModel` 增加 `storeAddressByStoreID` 回填映射。
  - 拉取预约列表后，额外按 `store_id` 拉店铺列表并用 `formattedAddress`（含 zip）回填地址。
  - `AppointmentDetailViewModel` 增加 `resolvedStoreAddress`，详情页加载时拉 `fetchStoreDetail` 获取完整地址。
  - 详情页与列表页地址展示统一支持多行，避免地址尾部被截断。

### 3) iOS：预约流程与设置页细节修复
- 文件：
  - `ios-app/NailsDashIOS/Sources/App/AppState.swift`
  - `ios-app/NailsDashIOS/Sources/Features/Appointments/BookAppointmentView.swift`
  - `ios-app/NailsDashIOS/Sources/Features/Home/HomeView.swift`
  - `ios-app/NailsDashIOS/Sources/Features/Stores/StoresListView.swift`
- 关键调整：
  - 增加 `dealsTabResetID` 与 `resetDealsNavigationStack()`，预约完成跳转后清理 Deals 残留路由。
  - 预约成功弹窗总金额文案改为 `+$` 结尾展示（如 `$48.00+`）。
  - `Pay at Salon` 文案固定为单行显示（不足时缩小字体），减少换行导致的视觉错位。
  - 设置页 `Privacy & Safety` 图标兜底处理，避免系统符号缺失导致空白图标。
  - 店铺列表默认排序保持 `Recommended`，移除拿到定位后自动切换为 `Nearest` 的逻辑。
