# 开发更新（2026-03-03）：iOS 写评论支持上传图片（对齐 H5）

## 本次目标
- iOS 端 `Transaction History -> Review` 写评论弹窗补齐图片上传能力。
- 交互与 H5 保持一致：最多 5 张、可预览/删除、先上传图片再提交评论。

## 代码改动

### 1) 评论上传图片接口接入
文件：`ios-app/NailsDashIOS/Sources/Features/Profile/ProfileRewardsService.swift`

- 新增上传 payload：`ReviewUploadImagePayload`。
- 新增方法：`uploadReviewImages(token:files:) async throws -> [String]`。
  - 调用后端：`POST /upload/images`
  - 请求类型：`multipart/form-data`
  - 字段名：`files`
- 新增上传错误解析：按 `detail` 解析后端错误，覆盖 `401/403/400/413/422` 等分支。

### 2) ViewModel 衔接上传与发评
文件：`ios-app/NailsDashIOS/Sources/Features/Profile/ProfileRewardsViewModels.swift`

- `OrderHistoryViewModel.createReview(...)` 增加 `images` 参数。
- 新增 `uploadReviewImages(token:files:)`，供页面层调用。

### 3) 写评论弹窗 UI 与逻辑补齐
文件：`ios-app/NailsDashIOS/Sources/Features/Profile/OrderHistoryView.swift`

- 新增状态：
  - 已有图片路径：`reviewExistingImagePaths`
  - 本地待上传图片：`reviewDraftImages`
  - 相册选择器项：`pickedReviewImages`
  - 上传中状态：`isUploadingReviewImages`
- 新增能力：
  - `PhotosPicker` 多选图片（最多 5 张）。
  - 图片预览网格（3 列）与单张删除。
  - 本地图片压缩与大小限制（单图上限 5MB）。
  - 提交时：先上传图片，成功后将 `images` 一并提交评论。
- 交互保护：
  - 上传中/提交中禁用提交按钮。
  - 超限、读取失败、上传失败给出明确弹窗提示。
