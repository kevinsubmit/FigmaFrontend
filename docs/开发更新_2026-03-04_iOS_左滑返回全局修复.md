# 开发更新（2026-03-04）：iOS 二级页面左滑返回全局修复

## 背景
- 需求：`Home / Book / Appointments / Deals / Profile` 5个主页面之外，其他二级页面都支持左滑返回上一页。
- 现象：仅启用系统 `interactivePopGestureRecognizer` 后，部分页面仍无法左滑返回。

## 根因
- 项目中大量页面使用了自定义导航栏（隐藏系统导航栏/自绘返回按钮），系统返回手势在某些页面生命周期中会被关闭或失效。
- 仅在根 `NavigationStack` 上开启一次手势，无法覆盖所有子页面和动态导航状态。

## 本次实现
### 1) 系统返回手势持续启用
- 文件：`ios-app/NailsDashIOS/Sources/App/NailsDashApp.swift`
- 新增全局安装逻辑：
  - 通过 `UIViewController.viewDidAppear` swizzle，在每个页面出现时重新启用：
    - `interactivePopGestureRecognizer.isEnabled = true`
    - `interactivePopGestureRecognizer.delegate = nil`

### 2) 增加左边缘拖动兜底手势（防系统手势被覆盖）
- 文件：`ios-app/NailsDashIOS/Sources/App/NailsDashApp.swift`
- 为 `UIViewController` 注入 `UIScreenEdgePanGestureRecognizer`（left edge）：
  - 当满足位移或速度阈值时执行回退。
  - 有导航栈可回退时执行 `pop`。
  - 模态页面时执行 `dismiss`。
- 作用：即使系统手势被某些自定义页面影响，二级页仍能左滑返回。

### 3) 启用入口保持
- 文件：
  - `ios-app/NailsDashIOS/Sources/Features/Home/HomeView.swift`
  - `ios-app/NailsDashIOS/Sources/Features/Auth/LoginView.swift`
- 保留 `.enableSwipeBackGesture()` 作为手势安装入口，确保全局机制被触发。

### 4) 通知页底部导航栏隐藏收口
- 文件：`ios-app/NailsDashIOS/Sources/Features/Profile/NotificationsModule.swift`
- 增加 `.toolbar(.hidden, for: .tabBar)`，避免进入通知页后出现底部主导航栏。

## 验证
- 本地编译验证：
  - `xcodebuild -project ios-app/NailsDashIOS.xcodeproj -scheme NailsDashIOS -configuration Debug -destination 'generic/platform=iOS Simulator' build`
  - 结果：`BUILD SUCCEEDED`

## 影响范围
- 仅 iOS 客户端 UI 导航交互层。
- 无后端接口、数据库、业务字段变更。
