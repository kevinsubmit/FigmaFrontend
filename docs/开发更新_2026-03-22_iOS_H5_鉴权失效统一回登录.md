# iOS / H5 鉴权失效统一回登录

## 问题
- 后端返回以下原始鉴权失败文案时：
  - `Could not validate credentials`
  - `token has expired`
  - `session expired`
  - `not authenticated`
  - `authentication required`
- Android 已统一按“会话过期，请重新登录”处理，并触发现有强制登出回登录页逻辑。
- iOS 与 H5 也需要补齐同样的边界识别，避免只有文案变化但没有触发回登录页。

## 本次调整
- iOS：补强 `AppState.shouldForceLogoutAfterSensitiveAuthAlert(...)`，新增识别上述鉴权失效关键词。
- H5：补强 `shouldForceRelogin(...)`，除 `401` 外，也会对上述鉴权失效文案直接触发强制重新登录。

## 影响文件
- `ios-app/NailsDashIOS/Sources/App/AppState.swift`
- `frontend/src/utils/authGuard.ts`

## 验证
- iOS：`xcodebuild -project ios-app/NailsDashIOS.xcodeproj -scheme NailsDashIOS -configuration Debug -destination 'generic/platform=iOS Simulator' CODE_SIGNING_ALLOWED=NO build`
- H5：`npm run build`
- 结果：均通过
