# 2026-03-03 H5 + iOS 网络超时策略统一

## 背景
此前 H5 与 iOS 请求超时策略不一致，部分请求等待时间过短或无明确超时提示，导致用户体验不稳定。

本次统一目标：
- H5 与 iOS 采用一致的超时分层策略
- 上传请求给更长超时窗口
- 超时后返回可读错误提示，避免“无响应”感

## 统一超时策略
- 读请求（GET）：15s
- 写请求（POST/PUT/PATCH/DELETE）：20s
- 上传请求（图片/头像/作品集等）：120s
- 刷新 token 请求：15s

## H5 改动
### 1) fetch 客户端（`frontend/src/api/client.ts`）
- 增加可选参数：`timeoutMs`
- 增加默认分层超时常量：`read/write/upload`
- 基于 `AbortController` 增加超时中止逻辑
- 超时错误统一为：
  - `Request timed out after Ns. Please try again.`

### 2) axios 客户端（`frontend/src/lib/api.ts`）
- 默认 timeout 从 10s 调整为读请求 15s
- request 拦截器中按请求类型动态设置 timeout
- 上传路径自动使用 120s
- refresh token 请求使用 15s

### 3) 错误文案映射（`frontend/src/utils/apiErrorMessages.ts`）
- 补充 timeout 文案映射：
  - `Request timed out. Please check your connection and try again.`

## iOS 改动
### 1) 统一 APIClient 超时策略（`ios-app/NailsDashIOS/Sources/Core/Network/APIClient.swift`）
- 新增统一常量：`read/write/upload/refresh`
- `URLSessionConfiguration` 对齐新策略
- 每个请求按 path + method 自动计算 `request.timeoutInterval`
- refresh token 请求单独设置 15s

### 2) 上传接口超时兜底
- `ProfileRewardsService.swift`
- `SettingsModule.swift`

改动：
- 上传 `URLRequest.timeoutInterval = 120`
- 捕获 `NSURLErrorTimedOut`，返回清晰错误文案

## 验证结果
- H5：`npm run build` 通过
- iOS：`xcodebuild ... build` 通过

## 影响范围
- 仅网络层超时策略与错误提示优化
- 不改业务接口
- 不改数据结构
- 不改权限逻辑

## 后续建议
- 可在 H5 和 iOS 增加“请求耗时埋点”，按接口统计 P95/P99
- 根据线上 RTT 再微调上传超时（120s -> 90s 或 150s）
