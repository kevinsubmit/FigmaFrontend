# H5 与后台登录错误提示修复

日期：2026-04-05

## 问题
- 用户输入错误账号或密码后，后端正常返回 `401 Incorrect phone number or password`。
- H5 页面没有展示后端错误文案，而是显示前端运行时错误：`Cannot convert undefined or null to object`。
- 后台管理端存在同类风险，错误头解析在某些场景下也可能二次抛错。

## 根因
- H5 `frontend/src/lib/api.ts` 在解析响应头时，取出了 `AxiosHeaders.get()` 方法后脱离原对象调用。
- `AxiosHeaders.get()` 依赖内部 `this` 上下文；401 错误链路触发该分支后，运行时在 axios 内部 `Object.keys(...)` 处报错。
- 后台管理端 `admin-dashboard/src/api/client.ts` 使用了同类调用方式，虽然当前未必每次触发，但存在相同隐患。

## 修复
- H5：对 `headers.get` 进行绑定后再调用，保留原始上下文。
- 后台管理端：使用 `headers.get.call(headers, ...)`，避免脱离上下文。

## 结果
- 错误密码登录时，H5 现在会正确展示：`Incorrect phone number or password.`
- 不再出现 `Cannot convert undefined or null to object`。
- 后台管理端同类错误头解析风险同步消除。

## 验证
- 后端实测：错误密码登录返回 `401` 与 `Incorrect phone number or password`。
- H5 运行态验证：错误密码登录页面显示正确文案。
- H5：`npm run build` 通过。
- 后台管理端：`npm run build` 通过。
