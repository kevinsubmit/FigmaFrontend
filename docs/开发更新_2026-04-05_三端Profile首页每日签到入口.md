# 三端 Profile 首页每日签到入口

日期：2026-04-05

## 变更目标
- 将每日签到入口从仅存在于 Points 页面，提升到三端 Profile 首页可见入口。
- 保持首页入口轻量，不引入 Home 页打扰。
- 签到成功后同步刷新积分余额、流水和 Profile 摘要。

## 实现范围
- iOS Profile 首页新增 Daily Check-In 卡片。
- Android Profile 首页新增 Daily Check-In 卡片。
- H5 Profile 首页新增 Daily Check-In 卡片。

## 交互规则
- 未签到：展示 `Check in now and earn 5 points.`，按钮显示 `+5 pts`。
- 签到中：按钮显示 `Checking...`。
- 已签到：展示 `Today's points already claimed.`，按钮显示 `Checked In` 并禁用。
- 签到成功后刷新 Points 余额、积分流水、Profile 页面 points 摘要。

## 具体实现
### iOS
- `ProfileCenterSummaryService.loadPrimarySummary(...)` 在加载 Profile 主摘要时附带读取每日签到状态。
- `ProfileCenterViewModel` 新增每日签到状态和签到动作。
- `ProfileCenterView` 在头像头部下方、VIP 卡片上方插入签到入口卡片。

### Android
- `ProfileCenterViewModel` 新增 `dailyCheckInStatus` 和 `claimDailyCheckIn(...)`。
- Profile 首页 `LazyColumn` 在 Header 和 VIP 卡片之间插入签到入口卡片。
- 签到成功后强制刷新 Profile 数据。

### H5
- `Profile` 页面加载资料摘要时附带读取每日签到状态。
- 新增 `handleDailyCheckIn`，成功后刷新 Profile 统计并广播 points 更新事件。
- 签到入口插入在 Profile 顶部信息区与 VIP 区块之间。

## 验证

- Android：`./gradlew :app:assembleDebug -x lint` 通过。
- H5：`npm run build` 通过。
