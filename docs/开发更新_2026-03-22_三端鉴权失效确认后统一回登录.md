# 开发更新 2026-03-22：三端鉴权失效确认后统一回登录

## 背景

当后端返回以下鉴权失效类文案时，三端之前的处理不完全一致：

- `Could not validate credentials`
- `token has expired`
- `session expired`
- `not authenticated`
- `authentication required`

存在的问题：

- Android 部分页面会弹出提示，但点击 `OK` 后不会统一跳转登录页。
- iOS / H5 已有部分全局处理，但原始后端文案与展示文案不完全统一，边界场景仍可能直接显示原始错误。

## 本次目标

统一 Android / iOS / H5：

1. 识别上述所有鉴权失效文案。
2. 统一展示为 `Session expired, please sign in again.`。
3. 用户点击 `OK` 后，清理当前会话并跳转登录页。

## Android

### 调整内容

- 在网络错误映射层把 `not authenticated` / `authentication required` 也统一归入会话过期。
- 在 `AppSessionViewModel` 中新增鉴权敏感文案归一方法，`forceLogout(...)` 时统一规范文案。
- 扩大 `shouldForceLogoutAfterSensitiveAuthAlert(...)` 匹配范围。
- 补齐 `Notifications` 和 `Profile` 子页弹窗确认后的登出跳转逻辑。

### 影响文件

- `android-app/app/src/main/java/com/nailsdash/android/core/network/NetworkError.kt`
- `android-app/app/src/main/java/com/nailsdash/android/ui/state/AppSessionViewModel.kt`
- `android-app/app/src/main/java/com/nailsdash/android/ui/screen/NotificationsScreen.kt`
- `android-app/app/src/main/java/com/nailsdash/android/ui/screen/ProfileDetailScreens.kt`

## iOS

### 调整内容

- 在 `AppState` 中新增敏感鉴权文案归一方法。
- `forceLogout(...)` 统一把上述原始后端文案规范为 `Session expired, please sign in again.`。
- `validation / server / network` 错误如果命中鉴权失效短语，也直接走登出回登录，而不是只停留在页面提示。

### 影响文件

- `ios-app/NailsDashIOS/Sources/App/AppState.swift`

## H5

### 调整内容

- 统一把 `not authenticated` / `authentication required` 映射为 `Session expired. Please sign in again.`。
- 无 token 的受保护请求也使用同一条文案，并走现有全局 relogin 弹窗。

### 影响文件

- `frontend/src/utils/apiErrorMessages.ts`
- `frontend/src/api/client.ts`

## 验证

- Android：`./gradlew :app:assembleDebug -x lint`
- iOS：`xcodebuild -project ios-app/NailsDashIOS.xcodeproj -scheme NailsDashIOS -configuration Debug -destination 'generic/platform=iOS Simulator' CODE_SIGNING_ALLOWED=NO build`
- H5：`npm run build`

验证结果：三端均通过构建。
