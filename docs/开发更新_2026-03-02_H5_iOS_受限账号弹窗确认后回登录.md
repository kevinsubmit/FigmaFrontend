# 开发更新（2026-03-02）：H5+iOS 受限账号弹窗确认后回登录

## 本次目标
统一 H5 与 iOS 在以下场景的行为：
- 未登录
- Session 过期
- 账号受限 / 封禁 / 风控拦截

统一规则：**先弹窗提示，用户点击 `OK` 后再清理会话并跳转登录页**。

---

## H5 端改动

### 1) 全局重登录事件机制
- 新增全局事件 `auth:force-relogin-required`。
- 新增 `completeRelogin()`：在确认后清理本地 token 并跳转 `/login`。

文件：
- `frontend/src/utils/authGuard.ts`

### 2) 全局顶层弹窗（Portal）
- 在 `AppRouter` 中监听全局重登录事件。
- 使用 `createPortal(..., document.body)` 挂载弹窗，确保层级最高。
- 弹窗打开时锁定 `body` 滚动与触控，禁止背景穿透点击。
- 弹窗样式加深遮罩、提升可见性，文案强制白色，按钮保留 `OK`。

文件：
- `frontend/src/App.tsx`

### 3) 网络层统一触发
- Axios 客户端：命中 401/受限消息后触发全局弹窗，点击 `OK` 后跳登录。
- Fetch 客户端：同样改为触发全局弹窗，不再直接无提示跳转。

文件：
- `frontend/src/lib/api.ts`
- `frontend/src/api/client.ts`

---

## iOS 端改动（并入本次提交）

### 1) 敏感鉴权消息统一判断
- 新增 `AppState.shouldForceLogoutAfterSensitiveAuthAlert(_:)`。
- 覆盖关键字：restricted / banned / blocked / suspended / unauthorized / not authenticated / session expired / sign in again。

文件：
- `ios-app/NailsDashIOS/Sources/App/AppState.swift`

### 2) 弹窗 OK 后强制退出登录
在以下页面 alert 的 `OK` 回调内加统一处理：命中敏感鉴权消息则 `forceLogout`。

文件：
- `ios-app/NailsDashIOS/Sources/Features/Appointments/BookAppointmentView.swift`
- `ios-app/NailsDashIOS/Sources/Features/Appointments/MyAppointmentsView.swift`
- `ios-app/NailsDashIOS/Sources/Features/Stores/StoreDetailView.swift`
- `ios-app/NailsDashIOS/Sources/Features/Stores/StoresListView.swift`
- `ios-app/NailsDashIOS/Sources/Features/Home/HomeView.swift`

### 3) 预约失败提示状态复位与限额提示
- 提交前清空旧的 success/error，避免旧错误卡住后续提交。
- 增加 daily booking limit 的标准化提示映射。

文件：
- `ios-app/NailsDashIOS/Sources/Features/Appointments/BookAppointmentViewModel.swift`

### 4) 图片详情页分享菜单新增下载
- 分享菜单增加 `Download image`。
- 申请/使用相册写入权限，保存成功/失败 toast 提示。

文件：
- `ios-app/NailsDashIOS/Sources/Features/Home/HomeView.swift`
- `ios-app/NailsDashIOS/Resources/Info.plist`
