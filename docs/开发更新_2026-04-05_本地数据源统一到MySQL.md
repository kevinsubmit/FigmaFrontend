# 本地数据源统一到 MySQL

更新时间：2026-04-05

## 背景

本地环境此前存在两套数据源：

- 活跃旧数据：`backend/nailsdash.db`（SQLite）
- Docker 后端活跃库：MySQL `nailsdash`

这会导致：

- App 端可能读到 SQLite 旧数据
- 后台管理页读到 MySQL 空库
- 同一台机器不同入口看到的数据不一致

## 本次处理

- 确认 Docker `backend` / `backend-scheduler` 统一使用 MySQL `nailsdash`
- 新增迁移脚本：`backend/migrate_sqlite_to_mysql.py`
- 将旧 SQLite 全量数据迁移到 MySQL
- 将本地 `backend/uploads` 同步到 Docker `/app/uploads`
- 清空 Redis 缓存并重启 `backend` / `backend-scheduler`
- 将本地 `backend/.env` 的 `DATABASE_URL` 切换为 MySQL
- 将活跃工作目录里的 `backend/nailsdash.db` 移出，只保留备份
