# 三端店铺作品详情页

日期：2026-03-26

## 本次改动
- iOS、Android、H5 的店铺详情页 `Portfolio` 图片改为可点击，进入独立的店铺作品详情页。
- 新详情页不复用 Home 的 pin detail，而是保持店铺上下文。
- 详情页统一为固定主图展示区，避免不同图片比例导致布局跳动。
- 详情页去掉店铺名、作品标题、作品描述，仅保留大图、计数、关闭/返回和底部 `Book Services`。
- 三端都去掉了底部缩略图。
- H5 详情页改为 portal 渲染，避免被店铺页收藏按钮和轮播点覆盖。
- H5 `Portfolio` 卡片点击结构改为单一原生 `button`，修复浏览器实际点击不触发的问题。

## 涉及文件
- `ios-app/NailsDashIOS/Sources/Features/Stores/StoreDetailView.swift`
- `android-app/app/src/main/java/com/nailsdash/android/ui/NailsDashApp.kt`
- `android-app/app/src/main/java/com/nailsdash/android/ui/screen/BookFlowScreens.kt`
- `android-app/app/src/main/java/com/nailsdash/android/ui/screen/StorePortfolioDetailScreen.kt`
- `frontend/src/components/StoreDetails.tsx`
- `frontend/src/components/StorePortfolioDetail.tsx`

## 验证
- Android: `./gradlew :app:assembleDebug -x lint`
- iOS: `xcodebuild -project ios-app/NailsDashIOS.xcodeproj -scheme NailsDashIOS -configuration Debug -destination 'generic/platform=iOS Simulator' CODE_SIGNING_ALLOWED=NO build`
- H5: `npm run build`

## 结果
- 三端都支持从店铺 `Portfolio` 进入作品详情页。
- H5 点击问题已在运行态确认修复。
