# 开发更新 - 2026-04-15 - 礼品卡转赠模板与领取预览

## 本次目标

为礼品卡转赠补上 v1 模板能力：

1. 发送人转赠时可选择礼品卡样式
2. 后端保存样式 key，避免三端各自硬编码结果不一致
3. 接收人在领取前即可看到礼品卡样式和留言预览
4. 领取后在 `My Gift Cards` 中继续看到同一张样式

## 后端改动

- `gift_cards` 新增 `template_key` 字段，默认值为 `minimal_gold`
- 新增静态模板服务 `app/services/gift_card_templates.py`
- 购买礼品卡时支持写入模板
- 转赠已有礼品卡时支持更新模板
- 新增接口：
  - `GET /api/v1/gift-cards/templates`
  - `POST /api/v1/gift-cards/claim-preview`
- `GiftCardResponse` 统一返回：
  - `template_key`
  - `template`
- 转赠回归脚本补充了模板字段和领取预览断言

## H5 改动

- `My Gift Cards` 列表卡片改为样式化礼品卡展示
- `Send this card` 弹窗新增模板选择器与大图预览
- `Claim Gift Card` 弹窗新增领取前卡面预览
- 领取成功后，卡片会按模板样式立即展示

## iOS 改动

- `GiftCardsView` 列表卡片改为样式化礼品卡展示
- 转赠 sheet 新增模板选择器与卡面预览
- 领取 sheet 新增领取前卡面预览
- `GiftCardsViewModel` 增加模板列表和领取预览状态管理

## Android 改动

- `My Gift Cards` 列表卡片改为样式化礼品卡展示
- 转赠 dialog 新增模板选择器与卡面预览
- 领取 dialog 新增领取前卡面预览
- `GiftCardsViewModel` 增加模板列表和领取预览状态管理

## v1 模板集合

当前内置模板：

- `minimal_gold`
- `birthday_bloom`
- `self_care_glow`
- `celebration_confetti`
- `thank_you_blossom`
- `midnight_luxe`

## 当前边界

- 模板仍是后端静态清单，不支持后台可视化管理
- 发送时可选模板，但不支持用户自定义上传封面
- 领取预览需要用户已登录且手机号与礼品卡接收手机号一致
- 这次没有扩展短信模板内容，仅保证三端内可见样式一致
